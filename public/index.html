3. Живой интерактив

В каждом режиме показываем одно и то же: что сделал клиент, что сделал сервер и что здесь важно заметить.

Unary RPC один запрос → один ответ

Самый простой сценарий. По ощущениям почти как обычный API-вызов, но внутри уже HTTP/2 и protobuf.

Нажми кнопку, чтобы увидеть сценарий по шагам.
Server streaming один запрос → много ответов

Клиент делает один вызов, а сервер потом присылает несколько обновлений в рамках того же соединения.

После запуска здесь появится понятная история потока.
Client streaming много сообщений → один итог

Клиент отправляет несколько частей, сервер ждёт завершения и только потом отдаёт итог.

Здесь будет разбор, как сервер собирает итог из нескольких частей.
Bidirectional streaming обмен в обе стороны

Обе стороны активны одновременно: клиент шлёт сообщения, сервер отвечает по ходу разговора.

После запуска покажем живой двусторонний обмен без путаницы.
