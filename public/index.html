3. Живой интерактив

        
Блоки открываются по мере прохождения: сначала unary, потом server streaming, потом client streaming, потом bidi.

      


      


      

        

          

            Unary RPC
            один запрос → один ответ
          

          
Самый понятный старт: один запрос пришёл, один ответ ушёл.

          
          
Сначала пройди простой unary выше или нажми эту кнопку.

        


        

          

            Server streaming
            один запрос → несколько ответов
          

          
Когда серверу удобно присылать обновления частями, не закрывая соединение.

          
          
Этот блок открылся после unary.

        


        

          

            Client streaming
            несколько сообщений → один итог
          

          
Клиент отправляет части, а сервер молчит до самого конца и потом даёт общий результат.

          
          
Этот блок откроется после server streaming.

        


        

          

            Bidirectional streaming
            обмен в обе стороны
          

          
Обе стороны могут говорить одновременно — это уже не просто «запрос, потом ответ».

          
          
Этот блок откроется после client streaming.
